# VERA 系统备份策略 (Backup Strategy)

为了保证代码的安全迭代以及避免不必要的大文件冗余，本系统采用以下标准化的备份策略：

## 1. 备份路径
所有的本地备份统一归档存放于项目根目录下的 `backups/` 文件夹中。

## 2. 备份排除内容 (Exclusions)
在执行本地文件备份或推送到代码仓库时，**必须严格排除**以下类型的文件和目录，以确保备份包的干净与轻量化：
*   **数据库文件**: 所有的 `.db` 文件及其相关日志（如 `vera.db`, `.db-shm`, `.db-wal` 格式）。
*   **运行日志**: 所有的 `.log` 格式文件。
*   **源数据与文档**: 所有的 `.csv` 数据表和 `.pdf` 文档。
*   **历史备份**: 必须排除 `backups/` 文件夹本身，防止发生无限嵌套备份。
*   **环境与缓存**: 虚拟环境依赖（如 `.venv/`）、Python 字节码缓存（`__pycache__/` 等）。

## 3. 时间戳命名 (Timestamping)
为了确保每次备份的独立性以及后期可追溯，每次本地备份动作所生成的备份文件夹都**必须增加精确到秒的时间戳**。
*   **时间戳标准格式**: `YYYYMMDD_HHMMSS` (例如：`20260221_104225`)
*   **命名示例**: `backups/VERA_PY_CLEAN_20260221_104225/`

## 4. 远程备份到 GitHub
除了在本地 `backups/` 目录进行留痕外，代码的进展还必须同步备份到远程的 GitHub 代码仓库中。
*   **远程官方仓库地址 (VERA Backend)**: [https://github.com/ZDZ66983270/vera](https://github.com/ZDZ66983270/vera)
*   仅提交 Python 源代码 (`*.py`)、配置文件、说明文档 (`*.md`) 以及其他构建项目必需的核心文件。
*   上述“备份排除内容”（如 `*.db`, `*.csv` 及 `backups/` 自身）**必须在项目根目录的 `.gitignore` 文件中明确声明**，杜绝大体积或敏感数据文件污染至 Git 的提交历史中。
